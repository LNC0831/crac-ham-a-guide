# 贡献指南

感谢你愿意让这套教程变得更好！这是一个社区驱动的开放教程，**任何形式的改进都欢迎**。

## 最需要的贡献（按价值排序）

1. **纠错** —— 发现讲解错误、答案错误、与最新官方文件冲突之处。这是最有价值的贡献，请尽量附上官方依据。
2. **把讲解讲得更透** —— 某个知识点你有更好的类比、更清晰的图示、更准确的原理解释。
3. **适配新版题库** —— 当 CRAC 发布新版题库时，更新题库数据与对应章节。
4. **补充真题精讲** —— 为某章补充更多"易错题逐题解析"。
5. **衍生形态** —— 做 Anki 卡组、网页版、PDF、B/C 类教程等（请遵守 CC BY-SA 4.0 的署名与相同方式共享）。

## 怎么提交

- **小修小补**（错别字、单点纠错）：直接提 Pull Request。
- **较大改动**（重写某节、调整结构）：建议先开 Issue 讨论，避免重复劳动。
- **报告问题但不改**：开 Issue，说明章节、原文、你认为的问题和依据。

## 写作风格约定（请尽量保持一致）

- **讲"为什么"，不堆"是什么"**：本教程的灵魂是解释原理，而非罗列答案。能推导的就推导，能类比的就类比。
- **中文为主**，术语首次出现时给出中英文对照。
- **标注不确定**：对没有十足把握的内容，明确写出"待核实"或注明来源，不要伪装成定论。
- **引用官方来源**：涉及法规、频率、数字的，尽量对齐官方原文（《无线电管理条例》《业余无线电台管理办法》67 号令、CRAC 题库等）。
- **准确性 > 完整性 > 文采**。与官方文件冲突时，一律以官方为准。
- 保持各章既有的排版风格：知识讲解 → 表格/对比 → 📝 真题精讲 → 📌 记忆卡片。

## 题库数据怎么更新

题库来源与版本记录在 [`题库/题库统计.md`](./题库/题库统计.md)。结构化数据为 `题库/A类题库.json`，字段：

```jsonc
{
  "id": "A-1",                       // 题号
  "codes": { "J": "LY0001", "P": "1.1.1" },  // J=官方题号, P=知识点章节编码
  "question": "题干……",
  "options": [ { "key": "A", "text": "选项内容" }, ... ],
  "answer_keys": ["A", "C"],         // 正确选项（可多个）
  "type": "single | multiple"        // 单选 / 多选
}
```

`题库/A类题库.md` 与 `_题库_第N章.md` 是由该 JSON 按 `codes.P` 分章生成的可读版本。更新题库时，请同步更新 JSON 与可读版，并在 `题库统计.md` 注明新版本号与来源链接。
